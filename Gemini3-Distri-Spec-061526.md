1. 系統概述與願景 (Executive Summary & Mission)
1.1 系統願景
隨著第三類高風險醫療器材（如主動式植入心律調節器、人工電子耳及骨科植入物）在全球與臺灣分銷鏈中的複雜度急劇上升，傳統以紙本或批次（Batch）更新為主的稽核模式，已無法應對近年頻發的「反向流向異常」、「灰色非授權跨境平行輸入（水貨）」以及「二級市場包裝防偽標籤竄改」等供應鏈欺詐行為。
本系統——「Remix: GeoEntity Master-061526 (臺灣醫療器材供應鏈 WGS-84 GIS 追蹤器與合規重組平台)」，是一套專為臺灣衛生福利部食品藥物管理署 (TFDA) 稽核專員與跨國醫材代理商設計的企業級主動式地理空間分析與智能合規監管引擎。系統整合了：
WGS-84 正投影二次維動態流向地圖核心，用於追蹤醫材在全臺分銷物流中心 (Distributor Hubs) 到各大醫學中心及教學醫院 (Medical Centers) 之間的地誌網絡。
基於 Gemini 3.5 認知多模態架構的「AI 智主重組與合規工作簿」，能將高度雜亂的多來源臨床稽核手稿 (Clinical Notes)、進口艙單與分銷報表，轉化為具備絕對法律與監管效力的合規驗證結構。
靈活的「TFDA 母機構地理空間名冊索引管理器」，支援 WGS-84 地理空間參考數據的動態解析、可視化編輯、批次 JSON 結構覆蓋、以及完整的上傳與導出。
本規格書將詳盡闡述本系統之底層架構、幾何投影演算法、資料模型、AI 核心算力架構，以及全新導入之高階 WOW AI 認知功能，旨在建立一套透明、可審計、且具备高維預測能力的防禦性供應鏈合規環境。

2.1 前端 React 19 / Tailwind 4 視圖層
動態狀態控制：由 React.useState 與 useRef 維護核心物流與院所狀態。前端狀態與地理空間管理器緊密綁定，確保任何名冊索引之修改均能即時同步至 SVG 地圖。
向量渲染引擎：屏棄第三方重型 GIS 庫（如 Leaflet 或 Mapbox）在沙盒容器內可能遭遇的渲染堵塞及 HTTPS 跨域載入阻礙，設計了輕量級、像素完美的 SVG 純數學二維投影映射系統，實現高頻流暢的客製化動畫（如光子飛躍流向線）。
自適應微交互：全面應用 Tailwind 4 提供的流體布局與 Motion 庫（motion/react）微動畫，使稽核員在進行多維度過濾（Slicers）時獲得低延遲的視覺反饋。
2.2 後端 Express / TSX 執行層
原生 TypeScript 支援：使用 tsx 在開發模式下無縫點火；生產環境經由 esbuild 快速封裝為單一的 dist/server.cjs 檔案，徹底消除 CommonJS 與 ES Module 對絕對路徑解析之相容性衝突。
Vite 中間件融合：在 process.env.NODE_ENV !== "production" 狀態下，動態掛載 Vite middlewareMode: true。這使所有程式碼在修改時不需重新配置端口，統一由內網 Nginx 代理映射至 3000 端口。

5.1 互動子分頁導航設計 (Three-Way Index Control Tab)
為避免傳統長表格產生操作眩暈，名冊管理器被精心隔離成三個互補的子維護控制台：
名冊總覽 (List View)：以緊湊的網格表格列出全臺 8 大醫管/樞紐機構代碼、官方全稱、機構類型、以及精確之經緯度座標。提供「修改」與「刪除」的即時前端重繪。
機構編輯器 (form View)：
主鍵校驗：若處於新增狀態，系統會自動在保存前檢索是否有重複的 entityId 衝突；
地理邊界安全閥門(Geofencing Guard)：緯度輸入限制於 20°N 到 26°N，經度限制於 118°E 到了 123°E。任何超出臺灣地理極限的經緯度數值，在點選保存時都會被 JavaScript 端提前攔截，彈出合規警告，防止流向線被投射至畫布外。
浮點數標準化：所有輸入皆強制經 parseFloat().toFixed(6) 格式化，確保 WGS-84 地球橢球體坐標精度達到了釐米級。
貼上與上傳維護面 (Bulk Text Area & Direct Drop)：
提供一個大容量文字輸入區，稽核專員能直接把 TFDA 資料庫導出的原始陣列 JSON 貼入。
檔案上傳介面：使用者能點選「選擇 WGS-84 檔案 (.json)」，利用瀏覽器 FileReader 異步讀取。系統會實施資料綱要檢驗 (JSON Schema Verification)，若陣列格式正確，會執行全量覆寫，這使得異地數據遷移不需要經由繁瑣的手動單條輸入。

6.1 已編碼之 9 大 WOW AI 認知功能細節與 Prompt 工程邏輯
1. 合規對帳清單生成器 (Compliance Checklist Generator)
Prompt 重載思維：促使 Gemini 依據傳入之當前點對點帳目流狀況，為現場稽核專員生成一份具備優先級的實地盤點清單。
邏輯核心：將未對帳的特定 SN（序列號）置於清單頂部，估計其「消失在供應鏈中」的機率。
2. 臨床流向差異標註器 (Clinical Discrepancy Spotter)
Prompt 重載思維：專注比對非結構化稽核筆記與報表中的數量落差。例如筆記中寫著「進口 50 箱骨釘」，但申報單只有「20 箱」，Gemini 會深入找出其中隱藏的「私自拼拆箱」合規風險。
3. 跨境條碼核實代理 (Cross-Border Verification Agent)
Prompt 重載思維：核對醫材條碼 UDI。針對 GS1-128 條碼語法、進口國家代碼進行對比，判定該產品是否有在歐洲流通卻在臺灣銷售之「平行輸入（灰產水貨）」特性。
4. 合規法規翻譯副駕駛 (Regulatory Translation Copilot)
Prompt 重載思維：將深奧複雜的 FDA 規範與臺灣《醫療器材管理法》法規進行交互核驗翻譯，並貼心將英文術語以 TFDA 官方傳統中文合規專有名詞進行對譯。
5. 高風險 SCM 模式估計器 (High-Risk SCM Pattern Estimator)
Prompt 重載思維：通過拓撲學分析，檢索是否有特定 Distributor 頻繁在多個子醫療院所之間「異常高頻率、小批量、深夜」移動醫材（常見的借調或串貨行為）。
6. 標準入庫與流向驗證器 (Standardized Inbound Validator)
Prompt 重載思維：針對入庫批次（Lot Number）提供高度嚴格的結構校正，將手寫日期（e.g.“端午節前進港”）轉化為 ISO-8601 時間戳記，並對過期風險提出早預警。
7. 不良事件危害批論風險矩陣 (Adverse Event Incident & Recall Matrix)
Prompt 重載思維：
系統定位：TFDA 臨床器械警示與不良反應分析專用。
推理邏輯：分析臨床手術事件記錄（如：心搏器電池除顫異常、骨板術後脆斷、醫用導管塗層剝落），將抱怨日誌中的關鍵字與分銷帳冊中的批號（LOT）進行地理空間交差核驗，判定受侵害之「高风险 LOT 與受害院所地圖矩陣」。
高亮標記：要求將受影響之 LOT 批號碼、院所英文簡稱 及 故障臨床代號 全數以 <span class="coral-highlight"> 渲染。
8. 防偽包裝主動稽核與安全認證器 (Graphic & Packaging Holographic Authenticator)
Prompt 重載思維：
系統定位：醫療產品防偽裝安全檢驗。
推理邏輯：審核紙盒外包裝封口防偽簽、條碼雷射折射紋、UV 螢光印記稽核紀錄。AI 透過外觀偏離徵兆分析，計算該醫材是否涉嫌「二次封裝（Expired Recycled Stock）」或「假標籤偽造」。
高亮標記：將 認證安全分數 (Authenticity Tool Score) 及 篡改風險信號 (Tamper Flag) 標記為高亮。
9. 跨境平行貿易與自訂關口審查器 (Custom Border Logistics Parallel Import Auditor)

在處理具備高度軍事保密價值的國軍總醫院、高敏感戰備醫療機構之 WGS-84 經緯度時，系統名冊模組內部支援隨機的
